These inputs set the sidebar_min_width and sidebar_max_width editor options, which clamp the sidebar width while the user drags the resize handle. The values are accepted as-is, but kept within reason:

  • min: if the user sets it below 280, the resize logic floors it to 280 — so the effective minimum is max(userMin, 280) and the sidebar can never be dragged below that.
  • max: used as-is for the drag clamp, but the CSS additionally caps the rendered width at 100cqw - 280px, guaranteeing the editor content area always keeps at least 280px of room. So a large max is allowed, just not always fully visible.
